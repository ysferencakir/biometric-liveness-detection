2. VERİ NE OLARAK KAYDEDILIYOR? (FORMAT)
================================================================================

VERİ YAPISI (DATA STRUCTURE):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📦 Temel Format: Python Dictionary (Sözlük)

Örnek:
------
{
    "Ahmet": [
        grayscale_image_1,  # NumPy array (garyscale pixelları)
        grayscale_image_2,  # NumPy array
        grayscale_image_3,  # NumPy array
        grayscale_image_4,  # NumPy array
        grayscale_image_5   # NumPy array
    ],
    "Fatma": [
        grayscale_image_1,
        grayscale_image_2,
        grayscale_image_3,
        grayscale_image_4,
        grayscale_image_5
    ],
    "Ali": [
        ... (5 tane grayscale image)
    ]
}

🔍 Her Kullanıcı İçin Depolanan Veriler:
────────────────────────────────────────

1. Anahtar (Key): Kullanıcı İsmi (String)
   • Örneğin: "Ahmet", "Fatma", "Ali"
   • Case-sensitive (büyük/küçük harfe duyarlı)
   • Boş olamaz

2. Değer (Value): Yüz Görüntüleri Listesi (List of numpy arrays)
   • Tam olarak 5 adet grayscale görüntü (gray image)
   • Her görüntü: 2D NumPy array
   • Veri tipi: uint8 (0-255 arasında pixel değerleri)
   • Boyutlar: Kamerada algılanan yüz çevresinin piksel boyutları
              (genellikle 100x100 veya 150x150 px arası)

📐 Grayscale Görüntü Format Detayı:
────────────────────────────────────

Orijinal kamera karesi (BGR renklendirmesi):
   ┌─────────────────────────────┐
   │ B=128  G=89   R=200         │  (Her piksel 3 channel)
   │ B=130  G=91   R=202         │
   │ ...                         │
   └─────────────────────────────┘

Grayscale'e dönüştürülmüş (Gri tonu):
   ┌──────────────────────┐
   │ 150                  │  (Her piksel 1 channel - gri ton)
   │ 152                  │
   │ ...                  │
   └──────────────────────┘

Kod:
────
gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
# frame: 3 channel renk görüntüsü (kamera çıkışı)
# gray: 1 channel grayscale görüntüsü

🎯 Neden Grayscale?
───────────────────
• Renk bilgisi yüz tanımayla ilgisiz (gri tonlar yeterli)
• Dosya boyutunu 3x küçültür
• İşleme hızını arttırır
• Histogram karşılaştırması daha etkili

📊 Bir Kullanıcının Toplam Veri Boyutu:
──────────────────────────────────────── 
• 1 grayscale görüntü: ~5-10 KB
• 5 görüntü: ~25-50 KB
• 10 kullanıcı: ~250-500 KB

KAYIT SÜRECİ (SAVE PROCESS):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Kod:
────
def save_database(db):
    with open(DB_PATH, "wb") as f:
        pickle.dump(db, f)

📝 Adım Adım İşlem:
───────────────────

ADIM 1: Dosya Açılması
   • "faces_db.pkl"  dosyası yazma modu ("wb") ile açılır
   • "w" = Write (yazma)
   • "b" = Binary (ikili format)
   • Eğer dosya yoksa: OLUŞTURULUR
   • Eğer dosya varsa: ÜZERINE YAZILIR (overwrite)

ADIM 2: Pickle Serileştirmesi
   • pickle.dump(db, f) çalıştırılır
   • Python dictionary → byte stream'e dönüştürülür
   • Tüm NumPy arrays, string'ler vs seri hale gelir
   
   Pickle nedir?
   ─────────────
   Pickle = Python Object Serialization Library
   • Python objelerini binary fıormata dönüştürür
   • Daha sonra geri yüklenebilir hale getirir
   • İnsan tarafından okunmaz (binary data)

ADIM 3: Diske Yazma
   • Byte stream doğrudan faces_db.pkl'ye yazılır
   • İşlem tamamlandıktan sonra otomatik kapatılır (with statement)
   • Dosya açık kalır mı? HAYIR (context manager otomatik kapatır)

🔄 OYut Akışı Diagram:
─────────────────────
   ┌──────────────────────┐
   │  Python Dictionary   │
   │ {                    │
   │  "Ahmet": [img...],  │
   │  "Ali": [img...]     │
   │ }                    │
   └──────────────────────┘
            ↓
   ┌──────────────────────┐
   │  pickle.dump()       │
   │  Serileştir          │
   └──────────────────────┘
            ↓
   ┌──────────────────────┐
   │  Binary byte stream  │
   │  \x80\x04\x95...     │
   │  (Okulamazız)        │
   └──────────────────────┘
            ↓
   ┌──────────────────────┐
   │  Diske yaz           │
   │ faces_db.pkl         │
   └──────────────────────┘

KALITSAL KONTROL:
─────────────────
• Herhangi bir exception durumunda catch edilir
• try-except bloğu hatar yakalar ve mesaj gösterir
• Dosya kısmi yazılmaz (atomic operation)

YÜKLE SÜRECİ (LOAD PROCESS):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Kod:
────
def load_database():
    if not os.path.exists(DB_PATH):
        return {}
    with open(DB_PATH, "rb") as f:
        return pickle.load(f)

📝 Adım Adım İşlem:
───────────────────

ADIM 1: Dosya Kontrolü
   • os.path.exists(DB_PATH) → faces_db.pkl varsa True, yoksa False
   • Dosya yoksa: boş dictionary {} döndürülür
   • Dosya varsa: dosya okunup yüklenir

ADIM 2: Dosya Açılması
   • "faces_db.pkl" dosyası okuma modu ("rb") ile açılır
   • "r" = Read (okuma)
   • "b" = Binary (ikili format)

ADIM 3: Pickle Deseri Haline Getirme
   • pickle.load(f) çalıştırılır
   • Byte stream → Python dictionary'ye dönüştürülür
   • NumPy arrays otomatik olarak geri oluşturulur

ADIM 4: Belleğe Yükleme
   • Database tüm verilerle RAM'e yüklenir
   • Dosya kapatılır (with statement)
   • Dictionary artık programda kullanılmaya hazır

🔄 Yükleme Akışı Diagram:
────────────────────────
   ┌──────────────────────┐
   │  faces_db.pkl        │
   │  Diskten dosya oku   │
   └──────────────────────┘
            ↓
   ┌──────────────────────┐
   │  Binary byte stream  │
   │  \x80\x04\x95...     │
   │  (Bellekte yüklü)    │
   └──────────────────────┘
            ↓
   ┌──────────────────────┐
   │  pickle.load()       │
   │  Deseri haline getir │
   └──────────────────────┘
            ↓
   ┌──────────────────────┐
   │  Python Dictionary   │
   │ {                    │
   │  "Ahmet": [img...],  │
   │  "Ali": [img...]     │
   │ }                    │
   └──────────────────────┘

ERY DURUMU:

• Dosya yoksa: {} (boş dictionary) döndürülür

5. KAMERA VE GÖRÜNTÜ İŞLEME
================================================================================

KAMERA AÇILIŞI:
───────────────

Kod:
    cap = cv2.VideoCapture(0)
    
Anlamı:
    • VideoCapture(): Video akışı yakalamak için objeyct
    • 0 = İlk kamera (default webcam)
    • 1, 2, 3... = Eğer birden fazla kamera varsa: USB kameralar
    
Kamera Kontrolü:
    if not cap.isOpened():
        print("Hata!")
    • Kamera bağlı mı? Sürücü yüklü mü? Kontrol eder

KARE OKUMA (FRAME CAPTURE):
────────────────────────────

Kod:
    ret, frame = cap.read()
    
Açıklama:
    • cap.read() → tuple döndürür: (ret, frame)
    
    ret = Boolean
        • True = Kare başarıyla okundu
        • False = Kare okunamadı (kamera bağlantısı kesildi?)
    
    frame = NumPy Array
        • Kameradan çekilen görüntü
        • Shape: (height, width, 3)
        • 3 = BGR channels (Blue, Green, Red)
        • Örnek: (480, 640, 3) = 480 pixel yükseklik, 640 genişlik
        • Veri tipi: uint8 (0-255 pixel değerleri)

GRIYE ÇEVIRME (GRAYSCALE CONVERSION):
──────────────────────────────────────

Kod:
    gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
    
Anlamı:
    • cvtColor() = Color space conversion (renk uzayı dönüşümü)
    • frame = Orijinal renkli görüntü (BGR)
    • cv2.COLOR_BGR2GRAY = Dönüşüm modu
    • gray = Sonuç (1 channel, sadece gri tonu)
    
Dönüşüm Formülü:
    Gray = 0.299 * R + 0.587 * G + 0.114 * B
    
    (İnsan gözü yeşile daha duyarlı, kırmızıya daha az)
    
Neden: Yüz tanımayla renk ilgisiz, gri tonlar yeterli

EKRANDA GÖSTERME (DISPLAY):
────────────────────────────

Kod:
    cv2.imshow("Pencere Adı", frame)
    
Anlamı:
    • "Pencere Adı" = Pencerenin başlık çubuğu metni
    • frame = Gösterilecek görüntü
    • Sonuç: Ekranda görüntü açılır
    
Pencere Kapatma:
    cv2.destroyAllWindows()
    • Tüm OpenCV pencerelerini kapatır
    • Bellekten temizler

TUS KONTROLÜ (KEYBOARD INPUT):
───────────────────────────────

Kod:
    key = cv2.waitKey(100)
    if key & 0xFF == ord("q"):
        break
    
Anlamı:
    • cv2.waitKey(100) = 100ms (0.1 saniye) Bekle
      - Milisaniye cinsinden zaman bekle
      - Bu süre içinde tuşa basılırsa key değeri döner
      - cv2.imshow() sonra çalışmalı
    
    • ord("q") = Karakterin ASCII değeri
      - "q" tuşunun sayısal değeri
    
    • key & 0xFF = Bit maskeleme yapan
      - Bazı sistemlerde key negatif değer döner
      - 0xFF ile AND işlemi yaparak sadece lower byte alındı
    
    • Sonuç: Q tuşuna basılırsa programa sinyal gönderilir

6. YÜZ ALGILA VE KARŞILAŞTIRMA
================================================================================

YÜZ ALGISLAMA (FACE DETECTION):
─────────────────────────────────

Kullanılan Metod: Haar Cascade Classifier
───────────────────────────────────────────

Kod:
    face_cascade = cv2.CascadeClassifier(
        cv2.data.haarcascades + "haarcascade_frontalface_default.xml"
    )
    faces = face_cascade.detectMultiScale(gray, 1.1, 4)

1. Haar Cascade Nedir?
   ────────────────────
   • Yüksek hızlı yüz algılama algoritması
   • 2001'de Paul Viola ve Michael Jones tarafından icat edildi
   • Önceden eğitilmiş bir model (pre-trained model)
   • haarcascade_frontalface_default.xml dosyası içinde depolanır
   • Binlerce görüntü kullanılarak eğitilmiş
   
   Nasıl çalışır?
   ──────────────
   • Yüzün özelliklerini keşfetti (gözler, burun, ağız vb)
   • Haar features kullanır (dikdörtgen desenler)
   • Görüntüyü taracak şekilde hareket eder
   • Her konumda "Bu yüz mü?" sorusunu sorar
   • Cascade = Sıralı filtreler (hızlı eleme için)

2. detectMultiScale() Detayları
   ─────────────────────────────
   
   Kod:
       faces = face_cascade.detectMultiScale(gray, 1.1, 4)
   
   Parametreler:
   ─────────────
   • gray = Input görüntü (grayscale)
   • 1.1 = scaleFactor
     - Görüntü ölçeği her iterasyonda 1.1x küçültülür
     - Farklı boyuttaki yüzleri algılamak için
     - Daha küçük = Daha yavaş ama daha doğru
     - Daha büyük = Daha hızlı ama eksik detektiyonlar
   
   • 4 = minNeighbors
     - Bir yüzü algılamak için gerekli komşu sayısı
     - Daha yüksek = Daha az yalan pozitif (false positive)
     - Daha düşük = Daha fazla algılama (bazı yalan)
     - 4 = dengelenmiş değer
   
   Dönen Değer:
   ────────────
   faces = [(x, y, w, h), (x, y, w, h), ...]
   
   List içinde tuples:
   • x = Dikdörtgenin sol üst köşesi (x koordinatı)
   • y = Dikdörtgenin sol üst köşesi (y koordinatı)
   • w = Genişlik (width)
   • h = Yükseklik (height)
   
   Örnek:
   faces = [(100, 50, 80, 100), (300, 120, 75, 95)]
   • Birinci yüz: x=100, y=50, w=80, h=100
   • İkinci yüz: x=300, y=120, w=75, h=95

3. Algılanan Yüzün Çıkarılması
   ─────────────────────────────
   
   Kod:
       (x, y, w, h) = faces[0]  # İlk yüzü al
       face_region = gray[y:y+h, x:x+w]  # Yüz alanını kırp
   
   NumPy Array Slicing:
   • gray[y:y+h, x:x+w]
   • gray: 2D array (grayscale görüntü)
   • [satır_başı:satır_sonu, sütun_başı:sütun_sonu]
   • y:y+h = y'den başlayıp h piksel boyunca (yükseklik)
   • x:x+w = x'den başlayıp w piksel boyunca (genişlik)
   
   Sonuç: Sadece yüz kısmı çıkarılır

================================================================================
HISTOGRAM KARŞILAŞTIRMASI (HISTOGRAM MATCHING):
─────────────────────────────────────────────────

Giriş Aşamasında Yüz Tanıması:
──────────────────────────────

Kod:
    hist1 = cv2.calcHist([face_region], [0], None, [256], [0, 256])
    hist2 = cv2.calcHist([stored_face], [0], None, [256], [0, 256])
    score = cv2.compareHist(hist1, hist2, cv2.HISTCMP_BHATTACHARYYA)

1. Histogram Nedir?
   ────────────────
   Pixel değerlerinin frekans dağılımı
   
   Örnek:
   Görüntüdeki tonlar:
   • 0 (siyah): 100 pixel
   • 50: 200 pixel
   • 100: 300 pixel
   • 150: 80 pixel
   • 200: 20 pixel
   • 255 (beyaz): 5 pixel
   
   Histogram = [100, 200, 300, 80, 20, 5, ...]
               (256 bin için gösterilir)
   
   Grafik:
   ──────
   Frekans
   ↑
   │    ┌─────┐
   │    │     │
   │    │     │ ┌─────┐
   │────┼─────┼─┼─────┼──→ Gri Tonlar
       0     100    200  255

2. İki Görüntü Histogramını Karşılaştırma
   ──────────────────────────────────────
   
   face_region = Kamerada algılanan yüz
   stored_face = Kaydedilmiş yüz
   
   Her ikisinin histogramı hesaplanır
   
3. cv2.compareHist() - Benzerlik Puanı
   ──────────────────────────────────────
   
   Yöntem: HISTCMP_BHATTACHARYYA
   
   Formül: d = √(1 - Σ(√(h1[i] * h2[i])))
   
   Basitçe:
   • Mit döner: 0 ile 1 arasında sayı
   • 0 = Tamamen aynı (perfect match / histogramlar aynı)
   • 1 = Tamamen farklı (no match / histogramlar çok farklı)
   • 0.5 = Kısmen benzer
   
   Bu programda:
   • score = karşılaştırma puanı
   • En yüksek score = En iyi eşleşme
   
   (Not: Bhattacharyya mesafesi için 0=benzer, 1=farklı)

4. En İyi Eşleşmeyi Bulma
   ──────────────────────
   
   Kod:
       best_match = "Unknown"
       best_score = 0
       
       for name, stored_faces in db.items():
           for stored_face in stored_faces:
               score = cv2.compareHist(hist1, hist2, ...)
               if score > best_score:
                   best_score = score
                   best_match = name
   
   Mantık:
   • Tüm kayıtlı yüzlerle karşılaştır
   • En yüksek puan (best_score) tutun
   • Hangi kişiye ait? (best_match) hatırla
   • Sonunda: En benzer yüzdeki kişi döner

5. Sonuç ve Renklendirme
   ────────────────────
   
   if best_match != "Unknown":
       color = (0, 255, 0)  # Yeşil dikdörtgen (başarılı eşleşme)
   else:
       color = (0, 0, 255)  # Kırmızı dikdörtgen (bilinmeyen yüz)
   
   Ekranda gösterilir

ÇİZİT HEDEFLERİ:

Hedef Kare Hızı (FPS): 30-60 FPS
Gecikme (Latency): < 100ms

PERFORMANS ÖZETİ:
───────────────────

İşlem                          Zaman        Kaynak
─────────────────────────────────────────────────────
1. Kameradan kare oku          ~30-50ms     GPU/CPU
2. Grayscale dönüş            ~10-20ms     CPU
3. Yüz algılama (detect)      ~50-100ms    CPU
4. Histogram hesaplama        ~20-30ms     CPU
5. Karşılaştırma (tüm DB)     ~100-500ms   CPU
6. Ekranda göster             ~10-20ms     GPU

Toplam: ~230-720ms per frame = 1-4 FPS

❌ DÜŞÜKtür! Çünkü:
─────────────────
• Histogram karşılaştırması: O(n*m) complexity
• Tüm kayıtlı yüzlerle karşılaştırılıyor
• Seri işleme (paralel değil)

İyileştirme Çalışmaları:
────────────────────────

1. Multi-threading
   • Histogram hesaplamayı paralel yap
   • ~2-3x hızlanma mümkün

2. Encoding-based Matching (FaceNet/ArcFace)
   • Histogram yerine encoding kulllan
   • Euclidean distance hesaplama hızlı
   • Daha sağlam eşleşme

3. Caching
   • Kaydedilmiş histogram'ları önceden hesapla
   • RAM'de sakla (hızlı erişim)

4. Batch Processing
   • Birden fazla kareyi toplu işle
   • GPU kullan (CUDA)

HAFIZA KULLANIMı (MEMORY USAGE):
─────────────────────────────────

5 kişi, her biri 5 fotoğraf (100x100 px):
• 1 grayscale görüntü: 100 * 100 * 1 byte = 10 KB
• 1 kullanıcı (5 fotoğraf): 50 KB
• 5 kullanıcı: 250 KB
• Python overhead: ~100-200 KB
• Toplam: ~400-500 KB
